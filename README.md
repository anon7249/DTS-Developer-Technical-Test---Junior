# DTS-Developer-Technical-Test

API Endpoints

The backend API is built using FastAPI and provides endpoints for creating, retrieving, updating and deleting caseworker tasks.

When the backend server is running locally, the API base URL is:

http://127.0.0.1:8000

FastAPI also provides automatically generated API documentation at:

http://127.0.0.1:8000/docs
Task Object

Each task contains the following fields:

Field	Type	Required	Description
id	integer	Generated by database	Unique ID for the task
title	string	Yes	The title of the task
description	string	No	Optional description of the task
status	string	Yes	Current status of the task
due_at	date/datetime	Yes	Due date or due date/time for the task

Example task:

{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}
Create a Task

Creates a new task and stores it in the PostgreSQL database.

Endpoint:

POST /api/create_task

Request body:

{
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}

Successful response:

{
  "Message": "Task created successfully",
  "task_id": 1
}
Retrieve All Tasks

Returns all tasks from the database, ordered by newest first.

Endpoint:

GET /api/get_tasks

Successful response:

[
  {
    "id": 1,
    "title": "Review case documents",
    "description": "Check all uploaded documents before the deadline",
    "status": "pending",
    "due_at": "2026-06-10"
  },
  {
    "id": 2,
    "title": "Contact applicant",
    "description": "Send follow-up email",
    "status": "done",
    "due_at": "2026-06-12"
  }
]
Retrieve a Task by ID

Returns a single task by its ID.

Endpoint:

GET /api/tasks/{task_id}

Example request:

GET /api/tasks/1

Successful response:

{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}

Error response:

If the task does not exist, the API returns:

{
  "detail": "Task not found"
}
Update Task Status

Updates the status of an existing task.

Endpoint:

PUT /api/tasks/{task_id}/status

Example request:

PUT /api/tasks/1/status?status=done

Successful response:

{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "done",
  "due_at": "2026-06-10"
}

Error response:

If the task does not exist, the API returns:

{
  "detail": "Task not found"
}
Delete a Task

Deletes a task from the database by its ID.

Endpoint:

DELETE /api/tasks/{task_id}

Example request:

DELETE /api/tasks/1

Successful response:

{
  "message": "Task deleted successfully",
  "task_id": 1
}

Error response:

If the task does not exist, the API returns:

{
  "detail": "Task not found"
}
Error Handling

The API includes error handling for common cases, including:

Returning a 404 Not Found response when a task cannot be found.
Returning a 500 Internal Server Error response if a task cannot be created.
Validating request data through the FastAPI/Pydantic schema before processing task creation.
