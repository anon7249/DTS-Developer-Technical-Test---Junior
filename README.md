# DTS-Developer-Technical-Test

## API Endpoints

The backend API is built with FastAPI and provides endpoints for creating, retrieving, updating and deleting caseworker tasks.

When running locally, the API base URL is:

```bash
http://127.0.0.1:8000
```

FastAPI also provides automatic API documentation at:

```bash
http://127.0.0.1:8000/docs
```

---

## Task Object

Each task contains the following fields:

| Field         | Type          | Required              | Description                            |
| ------------- | ------------- | --------------------- | -------------------------------------- |
| `id`          | integer       | Generated by database | Unique ID for the task                 |
| `title`       | string        | Yes                   | Task title                             |
| `description` | string        | No                    | Optional task description              |
| `status`      | string        | Yes                   | Current status of the task             |
| `due_at`      | date/datetime | Yes                   | Due date or due date/time for the task |

Example task:

```json
{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}
```

---

## Create a Task

Creates a new task and stores it in the PostgreSQL database.

```http
POST /api/create_task
```

### Request body

```json
{
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}
```

### Successful response

```json
{
  "Message": "Task created successfully",
  "task_id": 1
}
```

---

## Retrieve All Tasks

Returns all tasks from the database, ordered by newest first.

```http
GET /api/get_tasks
```

### Successful response

```json
[
  {
    "id": 1,
    "title": "Review case documents",
    "description": "Check all uploaded documents before the deadline",
    "status": "pending",
    "due_at": "2026-06-10"
  }
]
```

---

## Retrieve a Task by ID

Returns a single task by its ID.

```http
GET /api/tasks/{task_id}
```

Example request:

```http
GET /api/tasks/1
```

### Successful response

```json
{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "pending",
  "due_at": "2026-06-10"
}
```

### Error response

```json
{
  "detail": "Task not found"
}
```

---

## Update Task Status

Updates the status of an existing task.

```http
PUT /api/tasks/{task_id}/status?status=done
```

Example request:

```http
PUT /api/tasks/1/status?status=done
```

### Successful response

```json
{
  "id": 1,
  "title": "Review case documents",
  "description": "Check all uploaded documents before the deadline",
  "status": "done",
  "due_at": "2026-06-10"
}
```

---

## Delete a Task

Deletes a task from the database by its ID.

```http
DELETE /api/tasks/{task_id}
```

Example request:

```http
DELETE /api/tasks/1
```

### Successful response

```json
{
  "message": "Task deleted successfully",
  "task_id": 1
}
```

### Error response

```json
{
  "detail": "Task not found"
}
```
